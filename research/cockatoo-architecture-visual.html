01
North Star

Four domains, cleanly separated.

The architecture splits into four explicit layers. Each one has different rules about who can write to it, who can read from it, and how mistakes are recovered. Mixing them is how language-learning products start lying to their users.

The app owns truth. The model proposes, explains, and generates practice.

— Architectural Principle
01 / TRUSTED

Curriculum

Versioned language packs and the item graph: words, chunks, patterns, sentence frames. Bundled, signed, schema-validated. Never silently mutated by the model.

02 / PERSONAL

Learner State

Per-language progress, review outcomes, scheduler state. Exposure unlocks review eligibility; review performance drives mastery — not the other way around.

03 / UNTRUSTED

Generation

LLM-generated artifacts — cloze cards, distractors, examples, tutor replies. Strict structured output. Validated against schema and item rules. Cached, retired when wrong.

04 / CONTACT

Surfaces

Safari encounter, Learn review, Library inspection, Tutor explanation. Every surface writes back to the same item-state record — no dead ends.

02
The Closed Learning Loop

Every feature feeds the same circle.

Hover cards, reviews, tutor chats, and ambient page replacements all write to one place: the per-item learning record. That single rule keeps the system honest. No dead ends, no shadow state.

03
Curriculum Item Taxonomy

From flat word list to typed item graph.

The current CurriculumWord assumes one source, one target, one language, one tier. It cannot stretch to chunks, patterns, or sentence frames — which is what serious learners need. The new model handles all four with the same infrastructure, but treats each one differently for replacement safety.

Word

Ambient · usually safe

Single lexical item with a stable mapping. High-frequency function words and adverbs are usually safe to replace inline.

and → und
but → pero
today → hoy

Chunk

Ambient · review-first

Multi-word phrase with stable meaning. Lives in review and tutor first; ambient replacement only when phrase match is exact and low-ambiguity.

of course → por supuesto
I think → je pense
there is → es gibt

Pattern

Review-only · never inline

Grammar construction. Never replaced as raw text. Powers cloze, contrast, recomposition, and tutor explanations instead.

DE: weil + V-final
ES: tener que + inf
FR: partitive

Sentence Frame

Production · roleplay

Reusable sentence skeleton for production practice. Dangerous as automatic replacement because it usually requires grammar changes.

I would like …
I am going to …
Can you …?
safe → inline
review only

Data Model

Curriculum truth lives apart from learner progress.

The single most important separation: trusted items on one side, per-learner state on the other. Exposure no longer means progress — exposure unlocks review eligibility. Review performance is what drives mastery and replacement priority.

trusted · versioned

CurriculumItem

What the item is. Identical for every learner. Validated, signed, bundled with the app.

06
LLM Integration Points

Five places the model touches the app — each with a clear contract.

The model never writes to canonical curriculum directly, never sees page text by default, and never alone decides that an item has been mastered. Every output gets schema-validated, cached with prompt/model version, and recorded as untrusted until proven safe.

1

Pack Builder

Generates candidate curriculum items offline. Developer tool, not app UI.
Bad translations become product truth.
Schema validation, second-pass critique, manual spot-check, versioned packs.
2

Exercise Generator

Creates review cards (cloze, distractors, recomposition) from trusted items.
Exercise has wrong answer or introduces unseen grammar.
Require source item IDs, known answer, allowed vocab level. Cache only validated artifacts.
3

Tutor

Answers learner questions with bounded feedback on the selected item.
Overexplaining, wrong correction, accidental page-data upload.
Bounded prompt, no automatic page text, short responses, one correction at a time.
4

Production Feedback

Evaluates short learner-written sentences against a rubric.
Inconsistent grading; over-claims mastery.
Rubric-based structured output. Combine with repeated successes before marking strong.
5

Candidate Expansion

Suggests what the learner should see next, in the background.
Curriculum drift — model adds things the app didn't sanction.
Existing pack IDs allowed immediately. New generated items quarantined until validated.
07
Safari Extension Strategy

Stay conservative. The extension is the trust surface.

The content script is where the privacy boundary lives. The rule for beta is: words and a few exact chunks only, one replacement per block, no model calls from inside the page, and only item IDs reported back to the app.

For beta

  • Replace only words and a small set of exact chunks.
  • Keep one replacement per block by default.
  • Keep sensitive form exclusions.
  • Keep page text inside the content script.
  • Report only item ID, host policy, and exposure increment.
  • Match in order: exact chunks → priority due words → new words → reinforcement.

Not for beta

  • Model calls from the content script.
  • Automatic sentence rewriting.
  • Live grammar transformation.
  • Hidden upload of page context.
  • Matching substrings inside managed tokens, editable text, code, or controls.
  • Phrase replacement before false-positive testing.

Paid Beta Scope

Keep · Add · Cut.

The smallest defensible shape to charge for. The biggest single change before beta is adding a real review state — exposure counts are useful telemetry but they are not learning. Everything that expands platform, model, or media scope gets cut.

Already works

Keep

  • Safari inline word replacement
  • One active language at a time
  • Native dashboard surface
  • Popup pause / site controls
  • Vocabulary library
  • Tutor (bounded to selected token)
  • Privacy boundary
  • Sensitive-form exclusions
New for beta

Add

  • OpenAI provider · Keychain key, model setting
  • Real review state · lastSeenAt, attempts, nextReviewAt
  • Learn screen · recognition multiple-choice
  • Honest vocabulary screen — drop fake recall %
  • Onboarding checklist + test tutor button
  • 5-language starter packs (~175 items each)
  • Signed/notarized beta build + feedback link
  • Cloze review + simple interval scheduler (wk 3-4)
Not in beta

Cut

  • Ollama from the main user path
  • Local model settings UI
  • Multi-language simultaneous
  • Live sentence rewriting
  • Native-app replacement outside Safari
  • Voice / pronunciation scoring
  • Imported content library, Anki export
  • Streaks, gamification, complex SRS

Engineering Migration Path

Eight steps from one-language word list to five-language item graph.

The migration is staged so old behaviour stays unbroken at every commit. The German starter pack moves into the new model first, with tests proving nothing changes for existing users. The remaining four languages and the review scheduler arrive after.

01
Generic curriculum types

Add SupportedLanguage, CurriculumItem, LanguagePack alongside legacy.

02
German pack from existing data

Map the current 12 words into the new model. Tests prove old behaviour unchanged.

03
Update learning state

Move from exposureCounts dict to itemStates map. One-way migration.

04
Update extension payload

Send unlocked CurriculumItem records instead of hard-coded starter words.

05
Add review scheduler

Create due queues, record attempts. Vocabulary stops deriving recall from exposure count.

06
Five language packs

Ship small consistent starters: ~80 words, ~30 chunks, ~15 patterns, ~20 frames each.

07
OpenAI exercise generation

Cloze and contrast review cards from trusted items. Cache with prompt/model metadata.

08
Production feedback

Short writing checks after recognition/cloze are stable. Rubric-based grading.

Risk Register

Known failure modes, planned mitigations.

The product is honest about what it doesn't yet do. Each risk below has a structural mitigation already in the architecture — not a "we'll handle it in support" mitigation.

Risk · pedagogical
Feels too small to charge for
Charge for early access, not a finished course. Make the beta promise clear. The app must feel polished even with small scope.
Risk · competitive
Users compare it to free Toucan
Position on Safari-native Mac experience, privacy, review state, and tutor. Don't sell word replacement as the whole value.
Risk · operational
OpenAI costs or abuse
BYO-key for earliest beta, or hosted proxy with strict daily caps. Cheap models for routine work, better model only when needed.
Risk · pedagogical
Calling exposure "recall"
Track exposure, recognition, recall, production separately. For beta, implement recognition only and name it honestly.
Risk · onboarding
Safari extension setup friction
Onboarding must be excellent. Setup over a few minutes = churn before value. Test Tutor button confirms key + provider live.
Risk · compatibility
Page replacement breaks sites
Guarantee article/document pages only. Keep web apps experimental. Make disable-site obvious and reversible.
Risk · quality
Bad translation in canonical pack
Packs are versioned. Generated artifacts separate from canonical items. User report. Remote disable in future manifest.
Risk · privacy
Page text leaks to OpenAI
No page text in default extension messages. Selected-text tutor must show what will be sent. Reviews use curriculum items only.
