# GitHub demo media

This directory is the capture checklist for the v0.1 GitHub presentation.
Do not commit real browsing history, usernames, private URLs, or personal
vocabulary. Use a clean Safari profile or a neutral public article.

## Primary asset

Record one silent 30–45 second `demo-v0.1.mp4` (or an optimized
`demo-v0.1.gif`) in this order:

1. Safari Settings → Extensions: Cockatoo enabled;
2. a highlighted swapped word on a neutral page;
3. compact hover card, then expanded details;
4. extension popup with connection and practice statistics;
5. **Practice words** opening the native practice surface;
6. one short practice interaction;
7. quick cuts of Overview, Library, and Settings.

Use a 16:10 crop around 1440×900 or 1280×800. Keep the pointer slow, pause
briefly on each state, and avoid showing the full desktop. GIFs should be
under roughly 10 MB; prefer MP4 when GitHub playback is acceptable.

## Still-image fallback

If video capture is awkward, provide these PNGs:

```text
01-safari-extension-enabled.png
02-web-swap-hover.png
03-extension-popup.png
04-practice.png
05-overview.png
06-library.png
07-settings.png
```

After the files are supplied, add a compact hero image/video and a three-image
feature row near the top of the root README. Do not add broken placeholders to
the public README before the assets exist.
